01

What it is

One prompt becomes a coordinated org of specialist agents (research, engineering, security, testing, docs) plus a Forge meta-agent. They attack each other's work (a skeptic and an adversary kill weak claims before you ever see them), leave evidence files on disk you can audit, and write the lesson to memory so the next session starts smarter. It's model-agnostic: a procedure, not a model.

03

Why it's different: "looks done" isn't "verified"

We asked a plain agent to build secure JWT auth. It did a genuinely good job: a sound token core (Argon2id, refresh rotation, reuse-detection, revocation) that passed all 19 of its own tests. re-forge gives credit where it's due.

Then its security team audited that same passing code, and its adversary refused to inflate the demo-convenience findings. It doesn't cry wolf.

Green tests are not secure. That gap ("looks done" vs "verified") is the product.
